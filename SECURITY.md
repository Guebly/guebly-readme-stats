# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Guebly ReadMe Stats, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, send a detailed report to:

```
contato@guebly.com.br
```

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fix or mitigation (optional)

We will acknowledge your report within **48 hours** and provide a timeline for a fix.

## Disclosure Policy

Once a vulnerability is confirmed:

1. We will assign it to a maintainer
2. A fix will be prepared and reviewed
3. A patched release will be deployed
4. A public disclosure will be made after the fix is live (coordinated with the reporter)

## Scope

This policy covers the Guebly ReadMe Stats API and its official deployment at `readme.stats.guebly.com.br`.

Self-hosted instances are outside the scope of this policy, but we encourage maintainers of forks to follow the same responsible disclosure practices.
